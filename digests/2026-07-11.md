## 1. [Networking and the Internet, from First Principles](https://fazamhd.com/mental-models/networking/)

来源：**Hacker News** · ⬆ **9** points · 💬 **1** comments

### 🔑 关键概念

- 离散符号与再生：用有限且预先约定的符号（如莫尔斯电码的点和划）表示信息，沿线路的中继器只需检测脉冲是否存在，并重新生成一个干净、完整的信号。这样噪声不会逐段累积，使消息跨越长距离仍然无误。
- 协议：通信双方事先约定的一套规则，规定了消息格式、信号含义和交互顺序。电报时代是莫尔斯电码表和操作员的应答约定，现代互联网中的 IP、TCP、DNS、TLS 本质上也是这类书面协议。
- 中继：在电报网络中，每隔一段距离设置的中继器接收衰减和混入噪声的信号，再生成一个全新的脉冲继续发送。这是实现远距离数字通信的核心技巧，后来演变为分组交换中的“存储转发”。
- 数字网络：电报传输的是离散符号而非连续声音波形，因此它已经是早期的数字网络。这一设计使信号再生成为可能，并奠定了现代计算机通信的基础。

### 📝 文章摘要

文章尝试从第一原理出发解释互联网的工作方式，开篇用19世纪电报网络作为起点。作者展示了离散符号加再生中继如何解决模拟信号随距离衰减的问题，并指出莫尔斯电码就是一种早期的“协议”。文章强调，互联网并非一次性设计的蓝图，每一层协议（如 IP、TCP、DNS、TLS）都是为修补某个具体限制而陆续添加的。全文旨在用统一的思维模型解释日常网络现象，如地址栏中的锁标、网页卡顿与海底光缆故障时的路由切换。由于正文在摘要过程中被截断，关于分组交换、现代协议栈和海底光缆等后续内容未能完整展示，现有信息不足以概括全文要点。

## 2. [Einstein's relativity rules chemical bonds in heavy elements, new research shows](https://www.brown.edu/news/2026-07-09/chemical-bonds-relativity)

来源：**Hacker News** · ⬆ **292** points · 💬 **114** comments

### 🔑 关键概念

- σ键与π键：σ键是沿原子核轴线“头对头”重叠形成的强键，π键是“肩并肩”侧向重叠形成的较弱键，教科书中的三重键通常由一个σ键和两个π键构成。
- 相对论效应与重元素：当原子核质量足够大时，轨道电子的速度会接近光速，此时必须用爱因斯坦的相对论来描述电子行为，核越重效应越显著。
- 自旋-轨道耦合：在相对论条件下，电子的自旋磁矩与轨道运动不再相互独立，这种耦合改变了电子相互作用的规则，导致σ与π键的严格区分被打破。
- 光电子能谱：一种用激光将分子中电子击出的技术，通过测量电子飞出距离判断其结合能，从而揭示化学键的真实结构。

### 📝 文章摘要

布朗大学研究人员在《科学》上发表了直接实验证据，证明在重元素中相对论效应会改变三重键的结构。他们利用光电子能谱分析了碳-铋分子，发现化学键并非教科书描述的一个σ键和两个π键，而是更像一个π键和两个杂化的σ-π键，σ与π的界限变得模糊。这一发现挑战了长久以来的化学键理论，可能促使相关教科书改写，并影响铋等重元素在新型太阳能电池、量子材料等领域的应用研究。

## 7. [I converted an Android app to a webpage](https://danq.me/2026/07/09/your-app-could-have-been-a-webpage/)

来源：**Lobsters**

### 🔑 关键概念

- 安卓应用逆向工程：通过拦截网络流量、获取应用与服务器交换的 JSON 数据，从而还原应用的核心功能，文中用 HTTP Toolkit 代理并抓取了指定应用的请求。
- HTTP 代理拦截：将模拟器流量路由到代理工具，解密并查看 HTTPS 请求，作者借此发现应用仅通过拼接用户名和密码访问一个固定 URL 获取所有行程数据。
- JSON 数据结构化呈现：应用返回的 JSON 包含行程数组、广告数组和文件引用数组，作者用 Ruby 脚本定期拉取 JSON 并生成不含广告和追踪的静态 HTML 页面。
- 网页替代应用的特性对比：网页可复制、打印、保存、添加书签、搜索、跨设备使用且更易实现无障碍，而该应用仅有追踪和广告两个“额外功能”，且安装包体积达 43 MB（下载额外内容后膨胀至 124 MB）。
- 渐进式网页与“应用文化”冲突：部分本应作为网页交付的内容被强行封装为 app，增加开发部署成本，仅向更少用户提供更少功能，文中转制后的网页仅 0.05 MB，可选图像共 35 MB，且无隐私侵入。

### 📝 文章摘要

作者因子女表演学校组织去迪士尼演出，被要求安装一个名为 Travelbound 的应用以查看行程和住宿信息。他认为这完全可以用网页替代，于是通过 Android 模拟器拦截该应用的网络请求，发现它仅通过拼接用户名和密码的 URL 获取一份 JSON 数据，其中包含行程、广告和文件链接。他写了一个定时运行的 Ruby 脚本，将该 JSON 转换为不含广告与追踪的静态 HTML 页面，并设置与原团队相同的密码保护分享给同行者。最终生成的网页仅 0.05 MB（可选图片 35 MB），远小于应用本体 43 MB（膨胀后 124 MB），却能提供复制、搜索、书签等更多功能，且不再上报 Google 账户数据或展示“灵感”广告。该案例凸显了当下将简单信息展示包装为应用带来的资源浪费与功能损失，并为开发者提供了用网页重获控制权的示范。
