## 5. [Quadrupling code performance with a "useless" if](https://purplesyringa.moe/blog/quadrupling-code-performance-with-a-useless-if/)

来源：**Lobsters**

### 🔑 关键概念

- 指令级并行：现代处理器可以同时执行多条无依赖关系的指令，即使这些指令来自同一循环的不同迭代。
- 数据依赖链：当后一条指令依赖前一条指令的结果时，该循环迭代无法并行执行，性能受限于内存访问延迟。
- 分支预测与投机执行：CPU 会猜测分支方向并投机执行后续指令；如果预测错误，会回滚并重新执行，这一机制在这里被用来人为打破依赖。
- volatile 转换：通过将变量声明或转换为 volatile，阻止编译器认为某段代码无效或可优化，从而保留依赖关系表面上的“断裂”，引导 CPU 进行期望的投机执行。
- 延迟绑定与吞吐量绑定：前者指循环性能受单个指令延迟限制；后者指性能受指令排放速率限制，当依赖被打破后循环可从延迟绑定转变为吞吐量绑定。

### 📝 文章摘要

作者在优化一个领域专用压缩器时，发现一个看似简单的查找循环因数据依赖（`j = next_j[i][j]`）而受限于内存访问延迟，无法充分利用指令级并行。为打破依赖链，作者故意插入一个“无用的 if 判断”，结合 `volatile` 类型转换，使 CPU 误预测分支并投机执行，从而将循环从延迟绑定转为吞吐量绑定。标题声称这一技巧可带来“四倍性能提升”，原文在合成基准测试和真实测试中的具体提速数据未完整给出，仅表明此项改动有效提升了循环速度。该优化利用了硬件分支预测特性，而非传统的消除分支，属于一种反直觉的优化思路。
