## 6. [Work In Progress Rust](https://blog.dureuill.net/articles/wip/)

来源：**Lobsters**

### 🔑 关键概念

- 延迟错误处理：在开发早期暂不实现完整的错误逻辑，通过技巧将编译时错误降级为编译警告，让开发者可以先聚焦于主要功能路径。
- unwrap 临时占位：在需要返回 Result 或 Option 的地方调用 `.unwrap()`，以 panic 为代价让代码先行编译通过，后续再替换为正确的错误处理。
- todo!() 宏：Rust 标准库提供的发散宏（会 panic），可作为表达式占位符绕开类型检查，适用于标记尚未实现的功能或分支，避免分心。
- 文档注释警告技巧：用 `///` 注释未解决的待办事项，让编译器产生“文档注释未记录任何内容”的警告，用警告提醒开发者回头处理，又不阻断编译。
- CI 中将警告转为错误：在持续集成环境中使用 `-D warnings` 编译标志，将一切警告升级为硬错误，确保带 `todo!()`、`unwrap` 占位等未完成代码不会被意外合入生产分支。

### 📝 文章摘要

文章介绍了在 Rust 开发中如何暂时推迟处理正确性问题，以便先专注于核心逻辑。作者指出，开发期间被立即要求实现每条错误路径会频繁打断工作流，于是展示了几种将编译错误降级为警告的技巧：用 `unwrap()` 暂代错误处理、用 `todo!()` 宏占位未完成代码、利用文档注释产生提醒警告等。这些方法让代码能够快速编译和测试，同时通过 CI 的 `-D warnings` 在合并前将警告转为错误，防止未完成代码进入生产环境。作者强调这些只是临时策略，不能替代代码审查和最终的正确实现，并提到自己为此开发了一个辅助库以简化流程。

## 10. [Newly discovered PamStealer isn't your typical macOS malware](https://arstechnica.com/security/2026/07/new-pamstealer-macos-malware-uses-clever-tradecraft-to-remain-stealthy/)

来源：**Ars Technica**

### 🔑 关键概念

- PAM（可插拔认证模块）：macOS 内置的认证框架，PamStealer 利用它在本机验证用户登录密码，确认正确后才将凭证外传。
- JXA（JavaScript for Automation）：一种允许用 JavaScript 调用原生 Objective-C API 的脚本技术，此处的 AppleScript 自包含一个 JXA 下载器，静默获取下一阶段载荷。
- com.apple.quarantine：macOS 的一种文件属性，会标记从互联网下载的可执行文件，并在运行时给出警告或限制，该恶意软件通过诱导用户执行特定快捷键绕过这一保护。
- 信息窃取器（Infostealer）：专门窃取密码、凭证等敏感数据的恶意软件类别，PamStealer 即属于此类，且展现出比常见 macOS 窃取器更高的隐蔽性。
- 磁盘映像（.dmg）：一种 macOS 软件分发格式，攻击者用它伪装成正常剪贴板管理器 Maccy，诱骗用户挂载并点击其中的恶意 AppleScript。

### 📝 文章摘要

安全公司 Jamf 的研究人员发现了一种名为 PamStealer 的新型 macOS 信息窃取器，它通过伪装的磁盘映像分发，第一阶段是手工打造的 AppleScript，打开后并非调用常见 Shell 工具，而是执行自包含的 JXA 下载器，用原生 Objective-C API 获取第二阶段载荷。第二阶段由 Rust 编写，利用 PAM 在本地验证用户密码，确认正确后才将凭证发送到攻击者控制的服务器，且会伪装成 Finder 进程、加密 C2 通信并延迟约四十分钟才弹出全磁盘访问请求，以降低被察觉的风险。该恶意软件还结合诱导按键绕过 macOS 的隔离属性，整体执行链较传统 Mac 窃取器更安静，说明针对 Mac 的窃密工具正在朝更隐蔽、更贴近系统原生实现的方向演进。
