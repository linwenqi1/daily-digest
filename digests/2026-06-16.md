## 1. [The time the x86 emulator team found code so bad they fixed it during emulation](https://devblogs.microsoft.com/oldnewthing/20260615-00/?p=112419)

来源：**Hacker News** · ⬆ **220** points · 💬 **68** comments

### 🔑 关键概念

- 二进制翻译（Binary Translation）：模拟器不逐条解释x86指令，而是将整个代码块动态转换成宿主处理器的本地指令，类似JIT编译器，从而大幅提升执行性能。
- 循环展开（Loop Unrolling）：编译器将循环体复制多次以减少分支开销，但此例中编译器将初始化64 KB数据的循环展开为65 536条独立“写字节”指令，导致代码体积暴涨至256 KB。
- 栈探测（Stack Probe）：在Windows环境下，扩张栈不能只修改栈指针，必须按顺序访问每个页面，否则会触发页错误或异常，因此正确初始化大块栈空间需要逐页探测。
- 兼容层热修复（Shim/Hotfix in Emulation）：操作系统或模拟层可检测到特定程序的已知问题模式，并在运行时

## 2. [A backdoor in a LinkedIn job offer](https://roman.pt/posts/linkedin-backdoor/)

来源：**Hacker News** · ⬆ **1132** points · 💬 **208** comments

### 🔑 关键概念

- npm prepare 脚本：npm 在 `npm install` 完成后自动执行的脚本，攻击者利用此机制在开发者安装依赖时立即触发隐藏的恶意代码。
- 供应链攻击：攻击者伪装成正常代码审查任务，诱骗开发者克隆仓库并执行安装命令，从而在本地运行后门代码。
- 后门（Backdoor）：隐藏在 `app/test/index.js` 中的恶意代码，通过拼接远程 URL 下载并执行服务器返回的任意命令。
- 身份冒充：攻击者冒用真实开发者的 Git 提交身份和知名记者的 LinkedIn 个人资料，增加招聘骗局的可信度。
- 隔离审查：作者使用一次性云服务器和配置为只读模式的 AI 代理，在不执行恶意代码的情况下快速识别出可疑文件。

### 📝 文章摘要

一名开发者收到 LinkedIn 虚假招聘信息，被要求审查一个 GitHub 仓库并“检查弃用的 Node 模块问题”。作者出于谨慎，在隔离的云服务器上用只读 AI 工具审查代码，迅速发现 `app/test/index.js` 中隐藏的后门：`npm install` 会通过 `prepare` 脚本自动执行该文件，拼接远程服务器地址并运行其返回的任意内容。攻击者冒用了真实开发者和知名艺术记者的身份以获取信任。作者向 GitHub 和 LinkedIn 举报后，相关仓库和招聘资料仍未下架，表明此类依托求职场景的供应链攻击日益隐蔽且平台响应滞后。

### 💬 社区讨论

讨论集中在攻击的普遍性与平台责任上。多位用户指出，LinkedIn 无法有效

## 7. [Iroh 1.0 - Dial Keys, not IPs](https://www.iroh.computer/blog/v1)

来源：**Lobsters**

### 🔑 关键概念

- 密钥寻址（Dial by Key）：使用设备自主生成并控制的密钥作为网络地址，代替传统的 IP 地址进行连接，使设备无论移动到何处都能被稳定拨叫。
- IP 地址的不可靠性：IP 地址可能因网络切换、防火墙、NAT 等因素随时变化或变得不可达，完全不受终端设备控制。
- QUIC 多路径与 NAT 穿透：利用 QUIC 协议在同一连接内维护多条路径，可在网络条件变化时热切换路径，并通过加密方式实现 NAT 穿透，使设备间建立端到端加密的直连。
- 自定义传输插件：支持将低功耗蓝牙（BLE）、LoRa、WiFi Aware 乃至 Tor 等底层传输技术，统一接入密钥寻址的抽象模型。
- 本地优先（local‑first）：在无互联网访问的情况下，仍能发现并连接同一局域网内的设备。

### 📝 文章摘要

Iroh 发布了 1.0 稳定大版本，该协议栈用设备生成的密钥取代 IP 地址作为网络地址，致力于让连接“默认安全且直连”。在过去 30 天里，Iroh 运营的公共中继已经创建了超过 2 亿个端点，并已应用于视频流、大语言模型训练、安全聊天等场景。新版本基于 QUIC 多路径和 NAT 穿透实现高效直接传输，通常 95% 以上数据无需经过云端中继，显著降低流量成本。同时，1.0 确立了电线协议和语言 API 的稳定性承诺，并正式提供 Rust、Python、Node.js、Swift、Kotlin 等多语言绑定，方便嵌入移动应用。

- AMD PRO 技术：面向商用和专业市场的处理器附加功能集合，包含 TSME 等安全和管理特性，以往曾被下放到部分消费级 CPU。
